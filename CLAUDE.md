# 프론트엔드 TDD 학습을 위한 프로젝트


## 절대 하지 말아야 할 것들
- 애매한 부분이 생기면 추측하지 말고 무조건 물어봐라.
- 작업 중간에 임의로 다른 방향으로 바꾸지 마라.
- 임의대로 이슈, PR을 만들지 마라.


## 반드시 해야할 것
- CRITICAL: 해당 프로젝트는 TDD에 필요한 주요 메서드를 학습하기 위함이다. 모든 로직과 메서드를 이해할 수 있도록 테스트 코드테스트 모든 코드 바로 위에 `/** */` 포맷으로 주석을 단다.
- 주석 내용은 이 줄이 "의미하는 것"과 "역할" 중점적으로 설명한다.
- 모든 주석은 최대 2줄 이내로 작성한다.
